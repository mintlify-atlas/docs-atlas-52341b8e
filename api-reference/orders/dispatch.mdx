---
title: 'Dispatch order'
description: 'Update the status of an order and mark whether the confirmation has been sent.'
---

`PUT /api/orders/{orderId}/dispatch`

Updates an order's status and sets whether the customer information has been sent. Use this endpoint to move an order through its lifecycle (e.g., from pending to dispatched).

<Note>
  This endpoint runs on the Edge runtime.
</Note>

## Path parameters

<ParamField path="orderId" type="number" required>
  The integer ID of the order to update.
</ParamField>

## Request body

<ParamField body="status_id" type="number" required>
  The ID of the new status to assign to the order.
</ParamField>

<ParamField body="is_info_sent" type="boolean" required>
  Set to `true` if order confirmation information has been sent to the customer.
</ParamField>

## Response

<ResponseField name="message" type="string">
  Confirmation message. Value is always `"Order dispatched successfully"` on success.
</ResponseField>

<ResponseField name="order" type="object">
  The updated order object.

  <Expandable title="properties">
    <ResponseField name="order.id" type="number">
      Order ID.
    </ResponseField>

    <ResponseField name="order.order_code" type="string">
      Unique order code in the format `ORD-BRI{timestamp}`.
    </ResponseField>

    <ResponseField name="order.customer_id" type="number | null">
      ID of the associated customer.
    </ResponseField>

    <ResponseField name="order.comments" type="string | null">
      Free-text comments on the order.
    </ResponseField>

    <ResponseField name="order.created_at" type="string">
      ISO 8601 timestamp of order creation.
    </ResponseField>

    <ResponseField name="order.status_id" type="number">
      The newly assigned status ID.
    </ResponseField>

    <ResponseField name="order.is_info_sent" type="boolean">
      The updated value of the `is_info_sent` flag.
    </ResponseField>
  </Expandable>
</ResponseField>

<Warning>
  If the `orderId` does not match any existing order, the endpoint returns a `500` error with `{ "error": "Failed to update order status" }`.
</Warning>

<RequestExample>

```bash cURL
curl --request PUT \
  --url https://your-domain.com/api/orders/42/dispatch \
  --header 'Content-Type: application/json' \
  --data '{
    "status_id": 2,
    "is_info_sent": true
  }'
```

</RequestExample>

<ResponseExample>

```json 200
{
  "message": "Order dispatched successfully",
  "order": {

    "order_code": "ORD-BRI1711234567890",

    "status_id": 2,

  }
```

</ResponseExample>
