---
title: Quickstart
description: Clone the repository, configure a database, and run Los Guayacos locally in under ten minutes.
---

This guide walks you through every step from a fresh machine to a running Los Guayacos instance with a connected database.

<Steps>

<Step title="Check prerequisites">

Make sure you have the following before you start:

- **Node.js 18 or higher** — check with `node -v`
- **pnpm** (recommended) — install with `npm install -g pnpm`, or use npm/yarn instead
- **A PostgreSQL database** — local or a cloud provider (Neon, Supabase, or Railway all work)

<Tip>
  [Neon](https://neon.tech) offers a free serverless Postgres tier and works with the `@prisma/adapter-neon` driver that ships with this project. It is the easiest option if you do not have a local PostgreSQL instance.
</Tip>

</Step>

<Step title="Clone the repository">

```bash
git clone https://github.com/eggarcia98/food-order-system.git
cd food-order-system
```

</Step>

<Step title="Install dependencies">

<CodeGroup>

```bash pnpm
pnpm install
```

```bash npm
npm install
```

```bash yarn
yarn install
```

</CodeGroup>

<Note>
  The project's `package.json` lists pnpm as the intended package manager. Using pnpm avoids phantom dependency issues and matches the lock file committed to the repository.
</Note>

</Step>

<Step title="Configure environment variables">

Create a `.env` file in the project root and add the following variables:

```env .env
DATABASE_URL="postgresql://username:password@localhost:5432/food_orders?schema=public"
AUTH_ENDPOINT="http://localhost:8080/api/v1/auth"
```

Replace the value with the connection string for your database:

| Provider | Where to find the connection string |
|---|---|
| **Local PostgreSQL** | `postgresql://username:password@localhost:5432/database_name` |
| **Neon** | Neon dashboard → your project → **Connection Details** → copy the connection string |
| **Supabase** | Project **Settings** → **Database** → **Connection String** → select **URI** |
| **Railway** | Your Railway project → PostgreSQL plugin → **Connect** tab → **Connection URL** |

<Warning>
  Never commit your `.env` file. It contains database credentials. Confirm `.env` is listed in `.gitignore` before pushing.
</Warning>

</Step>

<Step title="Initialize the database">

Run the following commands in order:

```bash
# Generate the Prisma client from lib/prisma/schema.prisma
npx prisma generate --schema ./lib

# Push the schema to your database (creates all tables)
npx prisma db push --schema ./lib
```

`prisma db push` applies your schema directly without creating migration files. It is the fastest way to get a working database during development.

<Tip>
  To inspect your data in a visual browser interface, run `npx prisma studio --schema ./lib` and open `http://localhost:5555`. This is useful for verifying that tables were created correctly or for seeding initial records.
</Tip>

</Step>

<Step title="Run the development server">

<CodeGroup>

```bash pnpm
pnpm dev
```

```bash npm
npm run dev
```

```bash yarn
yarn dev
```

</CodeGroup>

The application starts at `http://localhost:3000`.

</Step>

<Step title="Place your first order">

Open `http://localhost:3000` in your browser. You will see the Los Guayacos landing page with two primary actions:

1. Click **Browse Menu** to see available dishes organised by nationality. Each dish shows its variants and prices.
2. Click **Start Ordering** to open the order creation form at `/new_order`.
3. In the order form, select a customer (or create one), choose dish variants, add any extras, and submit.
4. The order is saved to your database and assigned a unique `order_code`.
5. From the orders dashboard, you can view the order status and dispatch it using the **Dispatch** action, which calls `PUT /api/orders/[orderId]/dispatch`.

<Note>
  The menu and nationality data are seeded by your database records. If the menu is empty, use Prisma Studio (`npx prisma studio`) to add a `Nationality`, a `MenuItems` record, and at least one `ItemVariant` to see them appear in the UI.
</Note>

</Step>

</Steps>
