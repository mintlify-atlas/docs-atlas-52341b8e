**Los Guayacos** is a full-stack food ordering platform for an authentic Ecuadorian restaurant. It gives restaurant staff a single interface to manage the menu, receive and track customer orders, and dispatch them — while letting customers browse dishes and place orders online.

## What problem it solves

Running a restaurant order system by hand — paper tickets, phone calls, or disconnected spreadsheets — leads to missed orders, incorrect extras, and no visibility into status. Los Guayacos centralises the entire flow:

- Customers browse the menu and select dishes, variants, and extras in one form.
- Staff see every incoming order in a dashboard and can dispatch them with a single action.
- Customer records are stored and linked to orders so repeat customers are recognised automatically.
- Authentication keeps the management interface private while the menu remains publicly browsable.

## Architecture overview

Los Guayacos follows the Next.js **App Router** convention. Every route is a folder under `app/`, and server-side data access goes through **API Route Handlers** (`app/api/...`) that call Prisma directly.

```
app/
├── page.tsx              # Landing page (client component)
├── layout.tsx            # Root layout — fonts, header, footer, metadata
├── menu/                 # Public menu browsing
├── new_order/            # Order creation flow
└── api/
    ├── auth/             # login, signUp
    ├── orders/           # create, list, dispatch
    ├── customers/
    ├── dishes/
    ├── nationalities/
    └── sides/

lib/
└── prisma/
    └── schema.prisma     # Single source of truth for all data models
```

**Prisma** generates a fully-typed client from `schema.prisma`. The schema defines eight models: `Customer`, `Order`, `OrderItems`, `OrderItemExtras`, `MenuItems`, `ItemVariant`, `MenuExtras`, `Nationality`, and `Status`. All cascade deletes and relations are declared in the schema, so the database enforces referential integrity.

The `@prisma/adapter-neon` package replaces the standard Prisma TCP connection with a WebSocket-based driver compatible with Edge runtimes, enabling deployment to Vercel Edge Functions or Cloudflare Pages without connection pool issues.
